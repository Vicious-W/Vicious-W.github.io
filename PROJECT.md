## 1. 这是什么
- **项目**：我的个人主页（个人网站）。
- **类型**：静态网站（纯前端，无后端、无数据库）。
- **背景**：这是仓库所有者的**第一个开发项目**。仓库由一个 GitHub 仓库拉取而来，已有初步开发。

## 2. 目标与愿景
- 做一个**具有强烈表现力 / 视觉冲击力 / 交互有趣**的个人主页。
- 表现力优先：动画、富交互、（可能的）3D 等前端效果是重点。
- 整体气质 / 用途方向：整体的颜色以蓝色为主；网站应用展现个人的个性与艺术追求，并具有有趣的交互。

## 3. 工作约定（给 agent 和协作者——本节最重要）
- **所有者不做网页开发，也不打算为这个项目学习网页开发。** 网页开发不在所有者的学习范围内。
- 因此：**实现细节全权交给 agent。** 所有者负责设定目标、判断最终的观感与效果；agent 负责所有技术选型与实现层面的决策。
- 在「静态网站」这个前提下，尽量把效果做得惊艳。
- 可以大胆改动——本项目由 git 管理，任何一步都能回滚。
- **每次有实质进展或做出决策后，请更新本文件**（第 5 节「当前状态」＋第 7 节日志），让下一个进来的人或 agent 不必重新摸索。

## 4. 技术决策

**已确定**
- 静态网站，无后端。
- 托管平台：GitHub Pages
- 设计方向 / 整体气质：颜色以蓝色为主；简约、高级、交互有趣；展现个性，具有几何、数学、自然美感。
**待定（开放问题）**
- 具体技术栈：目前的开发只用到了 HTML/CSS/JS。后续会考虑更多技术，比如用静态框架（如 Astro 等）？动画 / 3D 用什么库？用什么技术栈由agent决定。

### 2026-06-23 — 修复第四屏本地与 GitHub Pages 视觉不一致
- 原因：第四屏 WebGL 先生成程序化动态云层，但 `sky-cumulus-natural.webp` 加载成功后会切换/混合为照片纹理；GitHub Pages 上该图片正常加载，因此显示成照片，本地 `file://` 直接打开时则更容易看到未被照片覆盖的程序化动态云层。
- 决策：第四屏以动态程序化云层为准，移除照片纹理加载与混合逻辑，并给 `scene.js` 引用加版本号，让 GitHub Pages、localhost、本地文件打开的视觉保持一致。

### 2026-06-23 — 首页重构为「极简开发者主页」＋ favicon
- 首页重构为单页四屏滚动的极简开发者主页（主题色 纸白→红→绿→蓝），衬线字体 + 蓝色点缀，WebGL（`scene.js`）+ 纹理背景；含 Hero / About / Selected work / Contact 四区块（见 commit `90e3038` v1、`e9251c2` v2）。
