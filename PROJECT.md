## 1. 这是什么
- **项目**：我的个人主页（个人网站）。
- **类型**：静态网站（纯前端，无后端、无数据库）。
- **背景**：这是仓库所有者的**第一个开发项目**。仓库由一个 GitHub 仓库拉取而来，已有初步开发。

## 2. 目标与愿景
- 做一个**具有强烈表现力 / 视觉冲击力 / 交互有趣**的个人主页。
- 表现力优先：动画、富交互、（可能的）3D 等前端效果是重点。
- 整体气质 / 用途方向：整体的颜色以蓝色为主；网站应用展现个人的个性与艺术追求，并具有有趣的交互。
- 第四屏水面场景的长期目标：追求**真实、自然、具有物理感**的夏日雨后小水塘。画面应呈现雷阵雨后蓝而透彻的天空、具有体积感的积雨云、水面对蓝天白云的自然倒影、可透过水面隐约看到的水草、小鱼，以及鼠标划过水面时产生的涟漪互动。总体目标不是简单装饰性背景，而是一个有真实水面质感、深度层次和生命感的实时视觉场景。

## 3. 工作约定（给 agent 和协作者——本节最重要）
- **所有者不做网页开发，也不打算为这个项目学习网页开发。** 网页开发不在所有者的学习范围内。
- 因此：**实现细节全权交给 agent。** 所有者负责设定目标、判断最终的观感与效果；agent 负责所有技术选型与实现层面的决策。
- 在「静态网站」这个前提下，尽量把效果做得惊艳。
- 可以大胆改动——本项目由 git 管理，任何一步都能回滚。
- **每次有实质进展或做出决策后，请更新本文件**（第 5 节「当前状态」＋第 7 节日志），让下一个进来的人或 agent 不必重新摸索。

## 4. 技术决策

**已确定**
- 静态网站，无后端。
- 托管平台：GitHub Pages
- 设计方向 / 整体气质：颜色以蓝色为主；简约、高级、交互有趣；展现个性，具有几何、数学、自然美感。
- 第四页水塘场景应升级为独立的高表现力实时图形模块。现有原生 WebGL shader 可以保留为原型参考，但长期目标需要更强的场景组织、材质、后处理、交互和资源管理能力。
- 第四页开发技术栈确定为 **Vite + Three.js + 原生 GLSL shader**：
  - **Vite**：负责开发服务器、模块化、依赖管理、资源打包与缓存控制；构建产物仍是静态文件，可部署到 GitHub Pages。
  - **Three.js**：负责第四页实时场景的场景图、相机、渲染器、纹理/模型资源、后处理与交互管线。
  - **GLSL shader**：负责核心视觉表现，包括水面法线、折射、反射、涟漪高度场、云层、水草摆动、鱼影/鱼群等。
  - **WebGL2 优先**：作为表现力与兼容性的基线。WebGPU 暂不作为主路线，以避免兼容性和工程复杂度过早上升。

**待定（开放问题）**
- 第四页最终采用完全程序化渲染、照片/视频纹理辅助，还是二者混合。原则上优先使用实时程序化效果保证互动与统一性，必要时使用少量高质量纹理增强真实感。

## 4.1 第四页开发

> 本节专门记录第四页（Contact / 水塘场景）的需求、问题边界与实现方案。凡是关于真实水面、蓝天白云倒影、水下水草、小鱼、鼠标涟漪和实时图形性能的讨论，都归入“第四页开发”。

**第四页的目标**
- 将当前第四页从“带动态倒影的背景”升级为“真实夏日雨后小水塘实时场景”。
- 画面预期：雷阵雨后，水塘倒影蓝天白云；天空蓝而透彻；云是具有体积感的积雨云；水面有真实波纹、微小扰动、高光、反射与折射；透过水面可隐约看到水下随水摆动的水草；偶尔有小鱼游过；鼠标划过水面时产生自然涟漪。
- 交互预期：鼠标/触控经过水面时影响水波，涟漪继续传播、衰减并影响倒影和水下折射。
- 真实感优先级高于短期实现速度；允许分阶段重建，不要求一次完成全部目标。

**当前第四页存在的问题**
- 旧实现曾是单个原生 WebGL fragment shader 加 CPU 波场贴图，适合轻量视觉原型，但不适合长期承载完整水塘场景；现已迁移到 Vite + Three.js + GLSL 的独立第四页模块。
- 已具备第一版真实水塘骨架：程序化蓝天/积雨云倒影、动态水面法线、反射/折射混合、水下水草、小鱼鱼影、焦散感和鼠标涟漪。
- 后续仍需继续深化真实感：云层体积感、真实水面材质层次、鱼和水草的自然运动、折射/高光/焦散的统一，以及更细致的性能调参。

**第四页分阶段方案**
1. 工程升级：迁移到 Vite，安装并引入 Three.js，保持现有四屏页面结构和 GitHub Pages 静态部署能力。
2. 场景骨架：把第四页 canvas 改为 Three.js 管理的实时场景，建立相机、渲染器、尺寸响应、可见性暂停、性能参数和基础后处理。
3. 真实水面第一版：实现水面平面、自定义 shader、动态法线、反射/折射混合、鼠标涟漪高度场，让水面先具备真实“水膜”和波动质感。
4. 天空与云：重建蓝天和积雨云表现，优先做高质量程序化天空与云层；必要时引入少量高质量纹理增强真实感。云层需要同时影响水面倒影。
5. 水下层：加入水草、鱼影/小鱼、深浅变化、水体雾化和水面折射，使“看见倒影”和“看见水下”同时成立。
6. 物理感与交互统一：让鼠标涟漪同时影响水面高光、倒影扭曲、水下折射，并让水草摆动和鱼的运动保持自然。
7. 性能与验收：用 Playwright MCP 在桌面和移动视口实际渲染检查，确认 canvas 非空、画面构图正确、互动有效、console 无报错，并根据帧率/视觉质量做调参。

**第四页验收标准**
- 第一眼能明确识别为雨后小水塘，而不是抽象蓝色纹理。
- 水面同时具有倒影、透明感、波纹、高光与轻微扰动。
- 天空和云看起来自然，云有体积感和层次，倒影中的云不会像贴图硬盖在水面上。
- 水下水草和小鱼可被察觉，但不过度抢戏；它们应被水体颜色、折射和波纹自然调和。
- 鼠标划过水面时涟漪自然传播并逐渐衰减，能真实影响画面。
- 页面仍是静态站，部署到 GitHub Pages 后不依赖后端。
